# Security - Review (Verbose)

Comprehensive security review of code and infrastructure.

> ⚠️ **DISCLAIMER:** This security analysis is generated by an AI assistant and is
> provided "as-is" without warranty of any kind. It is not a substitute for a
> professional security audit or penetration test. Findings should be validated
> by qualified security professionals. You are solely responsible for verifying
> recommendations and ensuring your code meets your organization's security requirements.

## Before Reviewing

Read the code. Do not produce findings from assumptions.
Ask what the code does if it is not clear — context changes severity dramatically.

If given a diff or PR: also read the surrounding code the diff touches, not just the changed lines.

## Review Categories

Work through all of these:

### 1. INJECTION

**SQL Injection:**
❌ **Bad:**
```python
query = f"SELECT * FROM users WHERE id = {user_id}"  # String interpolation
db.execute(query)
```

✅ **Good:**
```python
query = "SELECT * FROM users WHERE id = ?"
db.execute(query, (user_id,))  # Parameterized query
```

**Command Injection:**
❌ **Bad:**
```python
os.system(f"ping {host}")  # User-controlled host
```

✅ **Good:**
```python
import subprocess
subprocess.run(["ping", "-c", "1", host], check=True, timeout=5)
```

**Path Traversal:**
❌ **Bad:**
```python
with open(f"uploads/{filename}", "r") as f:  # filename could be "../../etc/passwd"
    return f.read()
```

✅ **Good:**
```python
import os
from pathlib import Path

base_dir = Path("uploads").resolve()
file_path = (base_dir / filename).resolve()

if not file_path.is_relative_to(base_dir):
    raise SecurityError("Path traversal attempt detected")

with open(file_path, "r") as f:
    return f.read()
```

### 2. AUTHENTICATION & SESSION

**Hardcoded Secrets:**
❌ **Bad:**
```python
API_KEY = "sk_live_abc123xyz"  # Hardcoded secret
```

✅ **Good:**
```python
import os
API_KEY = os.getenv("API_KEY")
if not API_KEY:
    raise ConfigError("API_KEY environment variable not set")
```

**JWT Issues:**
❌ **Bad:**
```python
# Algorithm confusion attack
payload = jwt.decode(token, SECRET_KEY, algorithms=["HS256", "none"])
```

✅ **Good:**
```python
try:
    payload = jwt.decode(
        token,
        SECRET_KEY,
        algorithms=["HS256"],  # Only allowed algorithm
        options={"require_exp": True}  # Require expiration
    )
except jwt.InvalidTokenError:
    raise AuthenticationError("Invalid token")
```

### 3. AUTHORIZATION

**Insecure Direct Object Reference (IDOR):**
❌ **Bad:**
```python
@app.route("/orders/<order_id>")
def get_order(order_id):
    # No check if current user owns this order!
    order = db.query(Order).get(order_id)
    return jsonify(order)
```

✅ **Good:**
```python
@app.route("/orders/<order_id>")
@login_required
def get_order(order_id):
    order = db.query(Order).get(order_id)
    if order.user_id != current_user.id:
        abort(403)  # Forbidden
    return jsonify(order)
```

**Mass Assignment:**
❌ **Bad:**
```python
user.update(**request.json)  # User can set any field, including 'is_admin'!
```

✅ **Good:**
```python
allowed_fields = {"name", "email", "phone"}
update_data = {k: v for k, v in request.json.items() if k in allowed_fields}
user.update(update_data)
```

### 4. CRYPTOGRAPHY

**Weak Algorithms:**
❌ **Bad:**
```python
import hashlib
password_hash = hashlib.md5(password.encode()).hexdigest()  # MD5 is broken
```

✅ **Good:**
```python
import bcrypt
password_hash = bcrypt.hashpw(password.encode(), bcrypt.gensalt())
```

**Hardcoded IV:**
❌ **Bad:**
```python
from Crypto.Cipher import AES
iv = b"1234567890123456"  # Hardcoded IV - breaks encryption
cipher = AES.new(key, AES.MODE_CBC, iv)
```

✅ **Good:**
```python
import os
from Crypto.Cipher import AES

iv = os.urandom(16)  # Random IV for each encryption
cipher = AES.new(key, AES.MODE_CBC, iv)
# Store IV with ciphertext: IV is not secret, just must be unique
```

### 5. DATA EXPOSURE

**PII in Logs:**
❌ **Bad:**
```python
logger.info(f"User login: {email} with password {password}")  # NEVER log passwords
```

✅ **Good:**
```python
logger.info(f"User login attempt: {email[:3]}***@{email.split('@')[1]}")  # Redacted
```

**Stack Traces Exposed:**
❌ **Bad:**
```python
@app.errorhandler(Exception)
def handle_error(e):
    return str(e), 500  # Exposes internal error details to client
```

✅ **Good:**
```python
@app.errorhandler(Exception)
def handle_error(e):
    logger.error(f"Internal error: {e}", exc_info=True)  # Log full details
    return {"error": "Internal server error"}, 500  # Generic message to client
```

### 6. INPUT VALIDATION

**Missing Length Limits:**
❌ **Bad:**
```python
name = request.form["name"]  # No length limit - could be 10MB string
```

✅ **Good:**
```python
name = request.form["name"]
if len(name) > 100:
    raise ValidationError("Name must be 100 characters or less")
```

**ReDoS (Regular Expression Denial of Service):**
❌ **Bad:**
```python
import re
# Catastrophic backtracking on input like "aaaaaaaaaaaaaaaaaaaaaaX"
pattern = re.compile(r"^(a+)+$")
if pattern.match(user_input):
    ...
```

✅ **Good:**
```python
import re
# Linear time regex
pattern = re.compile(r"^a+$")
if pattern.match(user_input):
    ...
```

### 8. INFRASTRUCTURE & CONFIGURATION

**Missing Security Headers:**
❌ **Bad:**
```python
@app.route("/")
def index():
    return "<html>...</html>"  # No security headers
```

✅ **Good:**
```python
@app.after_request
def set_security_headers(response):
    response.headers['X-Content-Type-Options'] = 'nosniff'
    response.headers['X-Frame-Options'] = 'DENY'
    response.headers['X-XSS-Protection'] = '1; mode=block'
    response.headers['Strict-Transport-Security'] = 'max-age=31536000; includeSubDomains'
    response.headers['Content-Security-Policy'] = "default-src 'self'"
    return response
```

**Overly Permissive CORS:**
❌ **Bad:**
```python
response.headers['Access-Control-Allow-Origin'] = '*'  # Allows any origin
```

✅ **Good:**
```python
ALLOWED_ORIGINS = ["https://app.example.com"]
origin = request.headers.get('Origin')
if origin in ALLOWED_ORIGINS:
    response.headers['Access-Control-Allow-Origin'] = origin
```

## Reporting Format

For each finding:

**Severity:** CRITICAL / HIGH / MEDIUM / LOW / INFO  
**Category:** (from the list above)  
**Location:** file name and line number or function name  
**What:** what the vulnerability is  
**Impact:** what an attacker could do if they exploited it  
**Fix:** concrete remediation with code example

---

## Severity Definitions

- **CRITICAL:** Exploitable with no authentication, data loss or full compromise possible
- **HIGH:** Exploitable with low effort, significant impact
- **MEDIUM:** Requires specific conditions, moderate impact
- **LOW:** Defense-in-depth issue, limited direct impact
- **INFO:** Best practice improvement, no direct exploitability
