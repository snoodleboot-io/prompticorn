The Problem

The AI tooling mess your team is living in

Whether your team has embraced AI agents or is still figuring it out, the same pattern emerges. Fragmentation, drift, and wasted hours nobody planned for.

Scenario A

Not using agents yet

Developers are using ChatGPT, Copilot autocomplete, or nothing structured. No shared AI workflow — each developer improvises. Onboarding new engineers means passing around .md files in Slack. Nobody knows what "the right way" to use AI in your codebase actually is. You're leaving serious velocity on the table.

Scenario B

Already using agents — but it's chaos

One dev uses Cursor, another uses Cline, another uses Claude Code. Each has their own agent config — written differently, named differently, maintained in isolation. A "code review agent" on one machine has nothing to do with the "review" agent on another. When someone new joins, they start from scratch. Config drift is invisible until something breaks.

Without Promptosaurus
  • Each developer configures their own AI agents from scratch
  • No shared standards — "code review" means something different to everyone
  • New hires spend days figuring out how the team uses AI
  • Switching tools means rewriting every config
  • Frontend and backend devs get the same bloated agent set
  • Configs drift silently — nobody knows what's current
With Promptosaurus
  • One config file, shared across the whole team
  • Consistent agent behavior for everyone, every tool
  • New hires run one command and they're ready
  • Switch tools in seconds — configs regenerate automatically
  • Persona filtering: each role gets exactly what they need
  • promptosaurus validate catches drift before it ships
How It Works

Three steps. Your whole team, aligned.

No build step. No CI pipeline changes. No DevOps work. Two minutes to set up. Done.

01

Initialize

Run promptosaurus init in your project. Answer a few questions about your team's tools, roles, and preferences. Takes two minutes. Promptosaurus generates a .promptosaurus.yaml config file.

02

Commit

Commit .promptosaurus.yaml to your repository. Now every developer on your team — current and future — has access to the same agent configuration. One file. Source of truth.

03

Generate

Each developer runs promptosaurus init locally. Their AI tool (Cursor, Cline, Claude Code, Copilot, Kilo) gets configured automatically — tailored to their role. Frontend engineers get frontend agents. Backend engineers get backend agents. Zero noise.

What promptosaurus init asks
  1. Which AI tools does your team use?
  2. What is your primary programming language?
  3. Which team roles (personas) need to be supported?

Output: configs generated for every selected tool, personas defined, .promptosaurus.yaml committed and ready to share.

Personas

The right agents for the right people

Not every developer needs every AI agent. A frontend engineer doesn't need Kubernetes debugging agents. A DevOps engineer doesn't need React pattern agents. Personas solve this.

When you set up Promptosaurus, you define which roles exist on your team. Each developer declares their persona. They get a focused, relevant agent set — nothing more, nothing less. Universal agents (ask, debug, explain) are always included for everyone.

⚛️

Frontend Engineer

React patterns, accessibility, CSS architecture, browser performance, component design systems, UI testing.

persona: frontend-software-engineer
⚙️

Backend Engineer

API design, database optimization, error handling, service architecture, background workers, data modeling.

persona: backend-software-engineer
🔧

DevOps / Platform

CI/CD pipelines, Kubernetes, infrastructure as code, security hardening, observability, incident response.

persona: devops-engineer
🧪

QA / Testing

Test strategy, coverage analysis, regression patterns, test automation, integration testing, quality gates.

persona: qa-tester
Personas are defined in .promptosaurus.yaml

Add roles, remove roles, or create custom ones for your team. Each developer runs promptosaurus swap to change their active persona without re-running the full setup.

YAML — .promptosaurus.yaml 
tool: claude          # cursor | cline | kilo-ide | copilot | claude
variant: minimal
language: typescript

personas:
  - frontend-software-engineer
  - backend-software-engineer
  - devops-engineer
  - qa-tester
Supported Tools

Works with the tools your team already uses

Select the tools your team uses during promptosaurus init. Promptosaurus generates the correct config files — automatically, in the right format, at the right paths.

Kilo
Kilo IDE

An AI-native IDE built for agentic development workflows.

Generates .kilo/agents/{agent}.md — YAML frontmatter + Markdown agent files, one per agent.
Cline
VS Code Extension

An autonomous AI coding agent that runs inside VS Code.

Generates .clinerules — a single drop-in Markdown rules file for Cline.
Claude Code
CLI

Anthropic's agentic coding CLI, used directly from the terminal.

Generates .claude/ — full directory of agents, workflows, and skills as Markdown files.
GitHub Copilot
GitHub

GitHub's AI pair programmer, integrated across VS Code, JetBrains, and more.

Generates .github/instructions/{agent}.md — version-controlled instruction files for Copilot.
Cursor
AI Editor

An AI-first code editor built on VS Code with deep agent integration.

Generates .cursor/rules/ + .cursorrules — rules files scoped per agent.
Switching tools

Already set up on Cursor but your team is moving to Claude Code? Run promptosaurus switch. Your config is preserved. New files are generated instantly for the new target. No rewriting. No drift.

Bash 
promptosaurus switch
# ? Which AI tool do you want to switch to?
#   > Claude Code
# Regenerating for Claude Code...
# Written: .claude/agents/code.md
# Written: .claude/agents/debug.md
# Written: .claude/agents/backend.md
# Done.

CLI Reference

Six commands. Complete lifecycle coverage.

Command What it does
promptosaurus init Interactive setup. Configures target tools, active persona, and output paths. Safe to re-run.
promptosaurus list Lists all registered agents available in the current project configuration.
promptosaurus switch Switches the active AI tool target. Rebuilds all output files for the new tool. One command.
promptosaurus swap Swaps the active persona. Re-filters the agent set and regenerates tool-specific configs.
promptosaurus update Regenerates configs from the current .promptosaurus.yaml. Non-destructive.
promptosaurus validate Validates the config and reports drift, missing files, or integrity issues before they ship.
